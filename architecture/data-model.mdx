Dev0 uses **Drizzle ORM** with **PostgreSQL**. The schema lives in `src/lib/db/schema.ts` and consists of four tables.

## Entity Relationship

```
projects
  │
  ├── tasks (many, cascades on delete)
  │     └── task_logs (one-to-one, cascades on delete)
  │
  └── sandboxes (many, cascades on delete)
```

- A **project** has many **tasks** and many **sandboxes**.
- Each **task** has at most one **task_logs** record (1-to-1, lazy-loaded).
- Each **sandbox** tracks which task it is currently assigned to via `taskId`.

export const projectStatusEnum = pgEnum('project_status', [

## Tables

### `projects`

The top-level entity. One project maps to one GitHub repository.

```typescript
export const projects = pgTable(
  'projects',
  {
    id: uuid('id').defaultRandom().primaryKey(),
    name: text('name').notNull(),
    description: text('description'),
    status: projectStatusEnum('status').default('PLANNING').notNull(),
    repoUrl: text('repo_url'),
    repoName: text('repo_name'),
    techStack: text('tech_stack').default('react-vite').notNull(),
    theme: text('theme').default('slate').notNull(),
    vibeInput: text('vibe_input'),
    specContent: text('spec_content'),
    createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
    updatedAt: timestamp('updated_at', { withTimezone: true }).defaultNow().$onUpdate(() => new Date()).notNull(),
  },
  (table) => [index('projects_status_idx').on(table.status)],
)
```

<ResponseField name="id" type="uuid" required>
  Primary key. Auto-generated UUID.
</ResponseField>

<ResponseField name="name" type="text" required>
  Human-readable project name generated by the preview agent.
</ResponseField>

<ResponseField name="description" type="text">
  One-line project description. Nullable.
</ResponseField>

<ResponseField name="status" type="project_status" required>
  Current lifecycle state. Default: `PLANNING`. Progresses through `READY → ACTIVE → COMPLETED`.
</ResponseField>

<ResponseField name="repoUrl" type="text">
  Full HTTPS URL of the GitHub repository (`https://github.com/{owner}/{name}.git`). Nullable until repo is created.
</ResponseField>

<ResponseField name="repoName" type="text">
  Short repository name (without owner). Used to build authenticated clone URLs. Nullable.
</ResponseField>

<ResponseField name="techStack" type="text" required>
  Template identifier. Default: `react-vite`. Supported values: `react-vite`, `tanstack-start`, `nextjs`.
</ResponseField>

<ResponseField name="theme" type="text" required>
  UI color theme token. Default: `slate`.
</ResponseField>

<ResponseField name="vibeInput" type="text">
  The original free-text product idea provided by the user. Nullable.
</ResponseField>

<ResponseField name="specContent" type="text">
  Structured specification generated by the planner. Injected into every task prompt. Nullable.
</ResponseField>

<ResponseField name="createdAt" type="timestamp with time zone" required>
  Row creation time. Default: `now()`.
</ResponseField>

<ResponseField name="updatedAt" type="timestamp with time zone" required>
  Last modification time. Auto-updated via `$onUpdate`.
</ResponseField>

**Indexes:** `projects_status_idx` on `status` (for listing active/ready projects).

### `tasks`

Individual units of work within a project. Each task maps to one execution run and one pull request.

```typescript
export const tasks = pgTable(
  'tasks',
  {
    id: uuid('id').defaultRandom().primaryKey(),
    projectId: uuid('project_id')
      .notNull()
      .references(() => projects.id, { onDelete: 'cascade' }),
    title: text('title').notNull(),
    description: text('description'),
    phase: integer('phase').default(1).notNull(),
    order: integer('order').default(0).notNull(),
    status: taskStatusEnum('status').default('PENDING').notNull(),
    geminiModel: geminiModelEnum('gemini_model')
      .default('gemini-3-pro-preview')
      .notNull(),
    dependencies: text('dependencies').array().notNull().default([]),
    prUrl: text('pr_url'),
    prNumber: integer('pr_number'),
    logs: jsonb('logs').$type<{}[] | null>(),
    attempts: integer('attempts').default(0).notNull(),
    maxAttempts: integer('max_attempts').default(3).notNull(),
    createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
    updatedAt: timestamp('updated_at', { withTimezone: true }).defaultNow().$onUpdate(() => new Date()).notNull(),
  },
  (table) => [
    index('tasks_project_idx').on(table.projectId),
    index('tasks_status_idx').on(table.status),
    index('tasks_project_phase_order_idx').on(table.projectId, table.phase, table.order),
  ],
)
```

<ResponseField name="id" type="uuid" required>
  Primary key. Also used as the Git branch suffix: `dev0/task-{id}`.
</ResponseField>

<ResponseField name="projectId" type="uuid" required>
  Foreign key → `projects.id`. Cascade deletes.
</ResponseField>

<ResponseField name="title" type="text" required>
  Short task title used in commit messages and PR titles.
</ResponseField>

<ResponseField name="description" type="text">
  Detailed task description injected into the agent prompt. Nullable.
</ResponseField>

<ResponseField name="phase" type="integer" required>
  Execution phase (1, 2, 3…). Tasks within the same phase can be parallelised in future; currently sequential. Default: `1`.
</ResponseField>

<ResponseField name="order" type="integer" required>
  Ordering within a phase. Lower order runs first. Default: `0`.
</ResponseField>

<ResponseField name="status" type="task_status" required>
  State machine status. Default: `PENDING`. See [Task State Machine](/architecture/task-state-machine).
</ResponseField>

<ResponseField name="geminiModel" type="gemini_model" required>
  Model used for this task's execution. Default: `gemini-3-pro-preview`. Flash tasks are lighter; Pro tasks are complex.
</ResponseField>

<ResponseField name="dependencies" type="text[]" required>
  Array of task UUIDs that must be `DONE` or `SKIPPED` before this task can run. Default: `[]`.
</ResponseField>

<ResponseField name="prUrl" type="text">
  GitHub PR URL set after successful `gh pr create`. Nullable until task reaches `REVIEW`.
</ResponseField>

<ResponseField name="prNumber" type="integer">
  GitHub PR number. Nullable.
</ResponseField>

<ResponseField name="logs" type="jsonb">
  Legacy inline log field. Execution logs are stored in the separate `task_logs` table. Nullable.
</ResponseField>

<ResponseField name="attempts" type="integer" required>
  Number of execution attempts so far. Incremented on each `FAILED` transition. Default: `0`.
</ResponseField>

<ResponseField name="maxAttempts" type="integer" required>
  Maximum allowed attempts before the task is permanently failed. Default: `3`.
</ResponseField>

**Indexes:**
- `tasks_project_idx` on `projectId`
- `tasks_status_idx` on `status`
- `tasks_project_phase_order_idx` on `(projectId, phase, order)` — the primary query path for dependency resolution

### `sandboxes`

Tracks E2B sandbox instances. One sandbox is provisioned per project and reused across sequential tasks.

```typescript
export const sandboxes = pgTable(
  'sandboxes',
  {
    id: uuid('id').defaultRandom().primaryKey(),
    projectId: uuid('project_id')
      .notNull()
      .references(() => projects.id, { onDelete: 'cascade' }),
    sandboxId: text('sandbox_id').notNull().unique(),
    status: sandboxStatusEnum('status').default('READY').notNull(),
    taskId: uuid('task_id').unique(),
    snapshotId: text('snapshot_id'),
    publicUrl: text('public_url'),
    createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
    updatedAt: timestamp('updated_at', { withTimezone: true }).defaultNow().$onUpdate(() => new Date()).notNull(),
  },
  (table) => [
    index('sandboxes_project_idx').on(table.projectId),
    index('sandboxes_status_idx').on(table.status),
  ],
)
```

<ResponseField name="id" type="uuid" required>
  Dev0 internal sandbox ID (used in API calls).
</ResponseField>

<ResponseField name="projectId" type="uuid" required>
  Foreign key → `projects.id`. Cascade deletes.
</ResponseField>

<ResponseField name="sandboxId" type="text" required>
  E2B's own sandbox identifier string. Unique constraint. Used to reconnect via `Sandbox.connect(sandboxId)`.
</ResponseField>

<ResponseField name="status" type="sandbox_status" required>
  Lifecycle status. Default: `READY`.
  - `READY` — provisioned and idle, can accept a new task
  - `RUNNING` — actively executing a task
  - `STOPPED` — killed or paused, cannot be reused
</ResponseField>

<ResponseField name="taskId" type="uuid">
  The task currently assigned to this sandbox. Unique constraint (one sandbox per task at a time). Nullable.
</ResponseField>

<ResponseField name="snapshotId" type="text">
  E2B template ID used to create the sandbox (`dev0-universal`). Nullable.
</ResponseField>

<ResponseField name="publicUrl" type="text">
  Preview URL exposed by E2B for the dev server port. Nullable.
</ResponseField>

### `task_logs`

Stores the full Gemini CLI event stream for a task execution. Kept in a separate table to avoid loading large JSONB payloads when listing tasks.

```typescript
export const taskLogs = pgTable(
  'task_logs',
  {
    id: uuid('id').defaultRandom().primaryKey(),
    taskId: uuid('task_id')
      .notNull()
      .references(() => tasks.id, { onDelete: 'cascade' })
      .unique(),
    events: jsonb('events')
      .$type<GeminiStreamEvent[]>()
      .notNull()
      .default([]),
    summary: text('summary'),
    totalTokens: integer('total_tokens'),
    durationMs: integer('duration_ms'),
    toolCallsCount: integer('tool_calls_count').default(0),
    createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
    updatedAt: timestamp('updated_at', { withTimezone: true }).defaultNow().$onUpdate(() => new Date()).notNull(),
  },
  (table) => [index('task_logs_task_id_idx').on(table.taskId)],
)
```

<ResponseField name="id" type="uuid" required>
  Primary key.
</ResponseField>

<ResponseField name="taskId" type="uuid" required>
  Foreign key → `tasks.id`. Unique — one log record per task. Cascade deletes.
</ResponseField>

<ResponseField name="events" type="jsonb (GeminiStreamEvent[])" required>
  Full ordered array of all `GeminiStreamEvent` objects captured during execution. Default: `[]`.
</ResponseField>

<ResponseField name="summary" type="text">
  Last full assistant message (truncated to 2000 chars). Used as a human-readable summary of what the agent did. Nullable.
</ResponseField>

<ResponseField name="totalTokens" type="integer">
  Total token count from the `result` event's `stats.total_tokens`. Nullable.
</ResponseField>

<ResponseField name="durationMs" type="integer">
  Execution duration in milliseconds, from `stats.duration_ms` or wall-clock fallback. Nullable.
</ResponseField>

<ResponseField name="toolCallsCount" type="integer">
  Number of `tool_use` events counted during the run. Default: `0`.
</ResponseField>

## Drizzle Relations

```typescript
export const projectRelations = relations(projects, ({ many }) => ({
  tasks: many(tasks),
  sandboxes: many(sandboxes),
}))

export const taskRelations = relations(tasks, ({ one }) => ({
  project: one(projects, { fields: [tasks.projectId], references: [projects.id] }),
  sandbox: one(sandboxes, { fields: [tasks.id], references: [sandboxes.taskId] }),
  executionLogs: one(taskLogs, { fields: [tasks.id], references: [taskLogs.taskId] }),
}))

export const sandboxRelations = relations(sandboxes, ({ one }) => ({
  project: one(projects, { fields: [sandboxes.projectId], references: [projects.id] }),
  task: one(tasks, { fields: [sandboxes.taskId], references: [tasks.id] }),
}))

export const taskLogsRelations = relations(taskLogs, ({ one }) => ({
  task: one(tasks, { fields: [taskLogs.taskId], references: [tasks.id] }),
}))
```

<Note>
  `task_logs` are lazy-loaded. When listing tasks, the `executionLogs` relation is not included. It is only joined when a user opens the task detail view, keeping list queries fast.
</Note>
